# План разработки приложения визуализации освещения

## Цель MVP
Сделать веб-приложение на React с:
- 2D-редактором на базе отрезков с выделением замкнутых контуров (граней),
- расстановкой столов и светильников на плане,
- 3D wireframe-визуализацией помещения и объектов (только просмотр),
- расчётом освещённости на поверхности столов,
- визуализацией результата (число/цветовая шкала).

---

## Предлагаемые упрощения для первой версии
1. Геометрия строится из отрезков в 2D; рабочими считаются только замкнутые контуры (грани).
2. Высота потолка одна на всю комнату.
3. Светильники пока только точечные изотропные:
   - параметры: позиция (x, y, z), угол вращения (храним, но не используем в расчёте),
   - типоразмер: только `lumens` из фиксированного набора `1000 | 2000 | 3000`.
4. Стол = прямоугольник на плане + высота рабочей поверхности `tableHeight`.
5. Стены для расчёта отражений не учитываем (прямой свет только).
6. Расчёт освещённости столов делаем дискретизацией по сетке точек на поверхности стола.
7. Редактирование выполняется только в 2D-плане (split-screen): слева редактор, справа 3D viewer.
8. Если нет корректного замкнутого контура с ролью `room`, расчёт и 3D-экструзия отключены.
9. В MVP допускается только один контур с ролью `room` (остальные контуры могут быть только `table`).

## Архитектура (черновая)

### Технологии
- React + TypeScript + Vite
- shadcn/ui (панели, формы, таблицы параметров)
- Three.js через React Three Fiber + drei
- Zustand (или Redux Toolkit) для состояния сцены

### Модули
1. **Scene Store**
   - room polygon, ceilingHeight
   - tables[]
   - luminaires[] (с `lumensPreset: 1000 | 2000 | 3000`)
   - selected entity / tool state
2. **2D Plan Editor**
   - canvas/overlay для рисования отрезков (ортогональные привязки)
   - инструменты: add segment, select segment/vertex/face, add luminaire
   - сетка и snapping по сетке
   - проверки размещения объектов внутри комнаты
   - подсветка отрезков, не вошедших ни в один замкнутый контур
   - выделение граней (surface/face) кликом внутрь полигона
   - drag вершины; если в точке клика совпадают две вершины, двигаются обе
3. **3D Viewer**
 - wireframe room extrusion
 - boxes for tables
 - point markers for luminaires
 - orbit/zoom/pan (только камера: вращение, зум, панорамирование; без выбора/редактирования объектов в 3D)
4. **Lighting Engine**
   - вычисление `lux` на точках столов
   - агрегации: min/avg/max по столу
   - защита от деления на ноль (`r >= epsilon`)
5. **UI Panels**
   - свойства выбранного объекта (face/segment/vertex/luminaire)
   - назначение роли выделенной грани: `room` или `table`
   - параметр высоты: `ceilingHeight` для `room`, `tableHeight` для `table`
   - список столов/светильников
   - кнопка пересчёта + legend

## Формула расчёта (MVP)
Для каждой точки стола `P`:
- от каждого светильника `L`:
  - `r = distance(L, P)`
  - `E = I / r^2`, где `I ≈ lumens / (4π)` для изотропного источника
- суммируем вклад всех светильников: `E_total = ΣE`

Примечания:
- Это упрощённая физическая модель для быстрого старта.
- Позже можно добавить косинус к нормали стола, ограничение cone angle и фотометрию IES.

## Важные уточнения (предлагаю зафиксировать)
1. Единицы координат: принять `1 unit = 1 meter`.
2. Диапазон высоты стола по умолчанию: `0.75 m`.
3. Высота светильника по умолчанию: `ceilingHeight - 0.1 m`.
4. Расчётная сетка стола по умолчанию: `10x10`.
5. Целевой диапазон для офиса (ориентир): `300–500 lux`.
6. Допустимые типоразмеры светильников (MVP): `1000`, `2000`, `3000` Лм.
7. Минимальный шаг сетки плана (по умолчанию): `0.1 m`.
8. Минимальная дистанция в расчёте: `epsilon = 0.1 m`.
9. Цвета статуса геометрии в редакторе:
   - отрезок в контуре: нейтральный,
   - отрезок вне контура: предупреждающий.
10. Роли граней MVP: только `room` и `table`.
11. Только один `room`-контур активен одновременно.
12. Порог слияния вершин (snap-distance): `0.15 m`.
